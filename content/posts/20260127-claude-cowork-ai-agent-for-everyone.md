Claude Code 爆紅之後，很多非工程師也想用，但一打開終端機就被嚇跑了。Anthropic 顯然注意到這個問題，於是推出了 Claude Cowork——一個讓任何人都能使用的 AI Agent 介面。

Boris 是 Claude Code 的創辦人，也是 Cowork 的共同創作者。他在這集 Podcast 中親自示範了 Cowork 能做什麼，也分享了 Anthropic 在安全設計上下的功夫。

## 不只是聊天機器人，是真的能幫你做事的 AI

Boris 開場就澄清了一個常見的誤解：很多產品自稱「AI Agent」，但其實只是聊天機器人加上一點搜尋功能。真正的 Agent 必須能「採取行動」——不只是產出文字，而是能操作你的電腦、控制你的瀏覽器、跟真實世界互動。

Cowork 就是這樣的工具。它的底層用的是跟 Claude Code 一模一樣的 Claude Agent SDK，只是包裝成一般人也能上手的介面。你不需要懂終端機、不需要會寫程式，只要用滑鼠點幾下、打幾個字，就能讓 AI 幫你做事。

## 實際示範：從整理收據到發送 Email

Boris 在節目中做了一個完整的示範。他在桌面上有一個收據資料夾，裡面有幾張亂命名的收據圖片。他給 Cowork 存取這個資料夾的權限，然後說：「幫我把這些檔案改名成收據上的日期。」

Cowork 開始工作。它讀取每張收據的內容、辨識日期、然後重新命名檔案。過程中有一張收據沒有日期，它不是自己亂猜，而是回來問 Boris：「這張沒有日期，要我跳過嗎？」

Boris 把這種行為叫做「Reverse Solicitation」——當 AI 不確定的時候，它會反過來問你，而不是自己做決定。這是刻意訓練出來的行為，目的是確保 AI 不會在不該行動的時候亂行動。

接下來 Boris 說：「把這些收據做成一個 Google Sheet。」Cowork 打開瀏覽器、登入 Google Sheets、建立新試算表、把資料一格一格填進去。過程中它發現貼上的格式有問題，就自己嘗試修正。

最後 Boris 說：「把這個試算表用 Email 寄給 Amy。」Cowork 打開 Gmail、在收件人欄位打「Amy」、從通訊錄中找到正確的人、附上試算表連結、寫好信件內容。一切就緒後，它問 Boris：「要寄出嗎？」

整個過程大概花了幾分鐘。有人可能會說：「這我自己做更快啊。」Boris 的回答是：沒錯，單一任務你做得更快。但關鍵是你可以同時跑很多個任務。當這個 Cowork 在處理收據的時候，你可以開另一個 Cowork 去做別的事。這是一種完全不同的工作方式。

## Anthropic 的安全設計：三層防護

讓 AI 控制你的電腦，聽起來有點可怕。Boris 花了不少時間解釋 Anthropic 在安全上做了哪些事。

第一層是模型本身的「對齊」（Alignment）。Anthropic 從成立第一天就是以 AI 安全為核心使命的公司。他們研究模型內部的神經元結構，用科學方法確保模型會按照人類意圖行動。這是所有安全措施的根基。

第二層是技術隔離。Cowork 底下其實跑著一個虛擬機，任何操作都在這個沙盒裡進行，不會直接影響你的系統。就算出了什麼問題，損害也被限制在可控範圍內。

第三層是行為防護。最近才加入的功能是「刪除保護」——當 Cowork 要刪除任何檔案時，它會先問你確認。另外針對 Prompt Injection（惡意指令注入）也有專門的防護，避免 AI 在瀏覽網頁時被惡意內容誤導。

Boris 坦言這些防護不可能完美，但這也是他們早期發布產品的原因之一：在實際使用中觀察問題，比在實驗室裡閉門造車更有效。

## Skills：讓 Cowork 學會用你的軟體

Cowork 有一個叫「Skills」的概念。當 Boris 請它做 Excel 試算表時，畫面上顯示它載入了一個 Excel skill。這是預先打包好的知識，教 Cowork 怎麼操作特定軟體。

如果你用的是冷門軟體——比如 AutoCAD、Salesforce、或是某個小眾的專業工具——你可以自己寫 skill，教 Cowork 怎麼用。這讓它的應用範圍幾乎沒有限制。

不過 Boris 建議初學者不要急著客製化。先用預設功能玩玩看，等到真的發現某個軟體它不太會用，再考慮寫 skill。保持簡單是上手的最快路徑。

## 2027 年會變成什麼樣？

主持人問 Boris：一年後人們會怎麼使用 Cowork？

Boris 笑說他只能用一週為單位來規劃，因為模型進步的速度太快了，任何長期預測都很容易失準。但如果硬要猜，他認為所有那些繁瑣的日常工作——把 A 軟體的資料搬到 B 軟體、在不同系統之間來回複製貼上、定期發送固定格式的報告——這些事情 AI 都會變得非常擅長。

他回顧了一年前的預測：當時他跟 Anthropic CEO Dario 都說過，到了年底人們可能不再需要自己寫程式。現在看來這個預測應驗了——Boris 自己最近兩個月的程式碼，百分之百是 Claude Code 寫的，他沒有親手寫過任何一行。

Boris 說，這種預測之所以能成真，是因為你必須相信「指數成長」。如果你只是把現在的體驗往前線性推估，你絕對不會相信 AI 能進步這麼快。但如果你真的把指數曲線畫出來，沿著曲線往前看，答案就在那裡。

## Cowork 第一週的成長超越 Claude Code

Boris 透露了一個有趣的數據：Cowork 上線第一週的成長速度，是當初 Claude Code 第一週的好幾倍。這顯示市場對「非工程師也能用的 AI Agent」有很強的需求。

他也觀察到一個有趣的現象：Anthropic 內部的非技術團隊——銷售、行銷、設計師、產品經理——幾乎每個人都在用 Claude Code。這讓他相信，降低進入門檻絕對是對的方向。

## 我的觀察

### 指數思維 vs 線性思維

Boris 在訪談中反覆提到「指數成長」這個概念，我認為這是整集最重要的訊息。

人類的大腦天生就是線性思考的。我們看到今天的狀況，會假設明天大概也差不多，後天也差不多。這種直覺在穩定的環境中通常是對的，但在 AI 領域完全不適用。

Boris 說他一年前預測「到年底人們不再需要自己寫程式」，當時很多人覺得誇張，因為當時的模型根本做不到。但他相信指數曲線，把曲線往前畫，答案就很清楚。結果他說對了。

這給我們一個啟示：當你在評估 AI 對自己工作的影響時，不要用現在的能力去推估未來。那條曲線比你想像的陡峭得多。一年前看起來不可能的事，現在已經在發生了；今天看起來不可能的事，一年後也很可能成真。

### 台灣工作者的機會

Cowork 的出現，對台灣的非技術工作者來說是一個很大的機會。

過去如果你想用 AI 自動化工作，你可能需要學 Python、學 API、學一堆技術名詞。現在這個門檻大幅降低了。你只需要會用滑鼠、會打字、會描述你想做什麼事，Cowork 就能幫你執行。

想像一下：行政人員可以自動整理報銷單據、業務可以自動更新 CRM 資料、行銷可以自動把社群數據整理成報表。這些過去需要工程師寫 script 才能自動化的工作，現在任何人都可以做到。

當然，這也代表競爭會變得更激烈。當每個人都能用 AI 提升效率時，「會用」就不再是優勢，「用得好」才是。現在正是投資學習的好時機——趁大多數人還在觀望的時候，先累積經驗，建立自己的工作流程。

Boris 說 Cowork 就像早期的 App Store：沒有人知道會長出什麼應用，但機會一定會出現。問題是，你準備好了嗎？
